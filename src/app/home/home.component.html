Welcome to the Bills Planner Application

This application is used to demonstrate full CRUD (create, read, update, delete) functionality within an Angular Front End.
To accompany this application there is also a fully enabled CRUD API Express back end.
Different components were used for each funtionality (outside of delete) to better understand Angular routing.


Below is a breakdown of the functionality in this application:

  • Get (All):
    		•  Used in the Bills Component with an ngFor to display every Bill
  • Get (One):
    		•  Used in the Edit and Patch Bill Componenets in conjunction with their functionality
  • Post:
    		•  Used in the Add Bill Component to post (add) newly created bill data to the back end
  • Put:
    		•  Used in the Edit Bill Component to put (edit) bill data in the back end
  • Patch:
    		•  Used in the Patch Bill Component to patch (change) only the paid value for the bill data
  • Delete:
    		•  Used in the Bills Component with the Get (All) to create a delete function per Bill

Here are some details on what the app includes:

  • Side Bar:
    		•  Allows the user to go between the different pages of the app
  • Home Page:
    		•  An explanation of the application and its functionalitys
  • Bills Page:
    		•  Displays a users current bills, prepopulated with mock data
  • Create Bill:
    		•  Button on bills page which shows fields used to create a new bill
  • Bill Order:
    		•  Selection on bills page used to reorder the bills based on the selected option
  • Edit Bill:
    		•  Button on each bill which brings the user to a new page to edit that bill
  • Delete Bill:
    		•  Button on each bill which will delete the bill entry entirely
  • Paid Button:
    		•  Radio Button on the edit bill page which changes (patch) the boolean value for paid
